# IAM User & Role Management with Least Privilege Access

## 📌 Project Summary
Implemented secure IAM user, group, and role configurations following the principle of least privilege to manage access across AWS services.

## 🏗 Architecture Overview
- IAM Users & Groups
- Custom IAM Policies
- IAM Roles for Service Access
- Policy Attachments & Boundaries

## 🚀 Key Features
- Role-based access control
- Custom policy creation
- Secure service-to-service permissions
- Principle of least privilege enforcement

## 🛠 Technologies Used
AWS IAM  
AWS Management Console  

## 📚 Key Learning Outcomes
- Identity and Access Management fundamentals
- Policy structure and JSON policy writing
- Role assumption workflows
- Security governance best practices

## 🔐 Security Considerations
- Restricted administrative access
- Scoped permissions per role
- MFA configuration

## 📈 Future Improvements
- Implement IAM Access Analyzer
- Integrate with AWS Organizations
