# Multi-Tenant Serverless File Storage System

## 📌 Project Summary
Designed a serverless file storage system supporting multiple tenants with logical isolation and scalable backend infrastructure.

This project focuses on multi-user architecture patterns in cloud environments.

## 🏗 Architecture Overview
- Amazon S3 (File Storage)
- AWS Lambda (Processing Logic)
- Amazon API Gateway (API Layer)
- Amazon DynamoDB (Tenant Metadata Storage)
- IAM for access management

## 🚀 Key Features
- Multi-tenant architecture
- Tenant-specific data isolation
- Secure file uploads
- Scalable serverless backend
- Metadata tracking using DynamoDB

## 🛠 Technologies Used
Amazon S3  
AWS Lambda  
Amazon API Gateway  
Amazon DynamoDB  
IAM  

## 📚 Key Learning Outcomes
- Multi-tenant architecture design
- Secure file handling in cloud environments
- Data partitioning strategies
- Access control enforcement

## 🔐 Security Considerations
- Tenant-based access controls
- Scoped IAM permissions
- S3 access restrictions
