# Infrastructure as Code (IaC) Automation using AWS CloudFormation

## 📌 Project Summary
Implemented Infrastructure as Code (IaC) using AWS CloudFormation to provision and manage cloud resources declaratively. This project eliminates manual configuration and ensures consistent, repeatable infrastructure deployments.

## 🏗 Architecture Overview
- CloudFormation Templates (YAML/JSON)
- Amazon EC2 provisioning
- Amazon S3 static website deployment
- Serverless REST API stack
- IAM roles and policies

## 🚀 Key Features
- Declarative infrastructure provisioning
- Reproducible environment setup
- Stack creation, update, and rollback support
- Parameterized templates for flexibility
- Automated resource dependency management

## 🛠 Technologies Used
AWS CloudFormation  
Amazon EC2  
Amazon S3  
AWS Lambda  
Amazon API Gateway  
IAM  

## 📚 Key Learning Outcomes
- Writing modular and reusable infrastructure templates
- Managing stack lifecycle operations
- Understanding dependency resolution
- Applying Infrastructure as Code best practices

## 🔐 Security Considerations
- Least privilege IAM roles defined in templates
- Controlled resource exposure
- Secure parameter handling

## 📈 Future Improvements
- Break templates into nested stacks
- Integrate with CI/CD pipeline
- Add drift detection automation
